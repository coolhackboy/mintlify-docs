# Welcome to PoYo AI API Documentation

PoYo AI provides state-of-the-art image and video generation APIs that enable developers to integrate advanced AI capabilities into their applications with ease.

## What is PoYo AI?

PoYo AI is a comprehensive API platform offering:

- **High-Quality Image Generation**: Create stunning images from text descriptions or transform existing images
- **Professional Video Generation**: Generate videos from text prompts or animate images
- **Multiple AI Models**: Choose from industry-leading models including GPT-4o, Sora 2, and VEO 3.1
- **Asynchronous Processing**: Efficient task management with real-time status updates
- **Simple Integration**: RESTful API design with comprehensive SDKs and examples

## How It Works

PoYo AI uses an asynchronous processing model for optimal performance:

<Steps>
  <Step title="Submit Task">
    Send a POST request to `/api/generate/submit` with your generation parameters
  </Step>
  <Step title="Receive Task ID">
    The API immediately returns a unique `task_id` in the response data wrapper
  </Step>
  <Step title="Poll Status">
    Query `/api/generate/status/{task_id}` to check progress (status: not_started, running, finished, or failed)
  </Step>
  <Step title="Retrieve Results">
    Once status is `finished`, the status endpoint returns your generated content in the `files` array
  </Step>
</Steps>

## Unified API Endpoints

PoYo AI uses a unified API structure for all generation types:

POST https://api.poyo.ai/api/generate/submit

Submit image or video generation requests.

### Query Task Status

```http
GET https://api.poyo.ai/api/generate/status/{task_id}
```

Check the status and retrieve results of a generation task.

## Quick Start Example

Here's a complete example of generating an image:

```python
import requests
import time

# Configuration
API_KEY = "your-api-key-here"
BASE_URL = "https://api.poyo.ai"
headers = {"Authorization": f"Bearer {API_KEY}"}

# Submit generation request
submit_response = requests.post(
    f"{BASE_URL}/api/generate/submit",
    json={
        "model": "gpt-4o-image",
        "callback_url": "https://your-domain.com/callback",
        "input": {
            "prompt": "A futuristic city skyline at night",
            "size": "1:1"
        }
    },
    headers=headers
)
task_id = submit_response.json()["data"]["task_id"]
print(f"Task submitted: {task_id}")

# Poll for results
while True:
    status_response = requests.get(
        f"{BASE_URL}/api/generate/status/{task_id}",
        headers=headers
    )
    task = status_response.json()["data"]

    if task["status"] == "finished":
        print(f"Image URL: {task['files'][0]['file_url']}")
        break
    elif task["status"] == "failed":
        print(f"Generation failed: {task['error']['message']}")
        break

    print(f"Progress: {task['progress']}%")
    time.sleep(2)
```

## Response Format

All API endpoints return JSON responses with a consistent structure wrapped in a standard envelope:

### Submit Response

```json
{
  "code": 200,
  "data": {
    "task_id": "task-unified-1757165031-uyujaw3d"
  }
}
```

### Status Response

```json
{
  "code": 200,
  "data": {
    "task_id": "task-unified-1757165031-uyujaw3d",
    "status": "finished",
    "progress": 100,
    "created_at": 1757165031,
    "completed_at": 1757165076,
    "model": "gpt-4o-image",
    "files": [
      {
        "file_url": "https://storage.poyo.ai/generated/image-abc123.jpg",
        "file_type": "image"
      }
    ]
  }
}
```

### Task Status Values

| Status | Description |
|--------|-------------|
| `not_started` | Task is queued and waiting to begin processing |
| `running` | Task is currently being processed |
| `finished` | Task completed successfully, results available in `files` array |
| `failed` | Task failed, error details available in `error` field |

### Error Response

```json
{
  "code": 400,
  "error": {
    "message": "Invalid parameter: size",
    "type": "invalid_request_error"
  }
}
```

## HTTP Status Codes

| Code | Description |
|------|-------------|
| 200 | Success - Request completed successfully |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 402 | Payment Required - Insufficient account balance |
| 403 | Forbidden - Access denied to requested resource |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error - Server error occurred |
| 502 | Bad Gateway - Service temporarily unavailable |
| 503 | Service Unavailable - Service is down for maintenance |
